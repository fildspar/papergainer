# PaperGainer 使用说明

> **no papers, no pain.**
>
> 一款纯静态前端工具，上传学术 PDF，AI 生成中文主要思路分析。无需服务器、无需注册，数据全部保存在本地浏览器中。

## 快速开始

1. **获取 MiniMax API Key**
   - 国际版：前往 [api.minimax.chat](https://api.minimax.chat) 注册并申请 API Key
   - 国内版：前往 [api.minimaxi.com](https://api.minimaxi.com) 注册

2. **配置 API**
   - 用浏览器直接打开 `index.html`（无需任何服务器）
   - 点击右上角 ⚙ 设置，填写 Base URL、API Key 和模型名称后保存

3. **分析论文**
   - 将 PDF 文件拖拽到上传区，或点击"点击选择文件"
   - 选择分析模板，点击"开始 AI 分析"
   - 分析完成后自动跳转到详情页展示结果

## 功能介绍

### 主页（index.html）

**PDF 上传分析**

- 支持拖拽或点击选择 PDF 文件，可同时上传多个（批量分析）
- 文件仅在本地浏览器内解析，不会上传到任何服务器
- PDF 文本提取最多读取前 25 页，发送给 AI 的内容截取前 14000 字符

**分析模板**

在上传区下方可选择五种内置模板：

| 模板 | 适用场景 |
|------|----------|
| ◈ 结构化学术 | 固定分节：研究背景 / 核心方法 / 实验设计 / 主要结论 / 局限性 |
| ◎ 要点速览 | 一句话总结 + 6-8 个核心要点，适合快速浏览 |
| ⇌ 对比分析 | 与已有工作的创新点对比，适合了解方法差异 |
| ◉ 通俗解读 | 用非专业语言解释，适合跨领域阅读 |
| ✦ 自定义 | 使用在设置页配置的自定义提示词 |

**论文获取导航**

页面下方汇聚了常用学术平台的快捷入口，分为三类：
- **学术搜索**：Google Scholar、arXiv、PubMed、Semantic Scholar、百度学术、ResearchGate
- **出版商数据库**：Nature、IEEE Xplore、ScienceDirect、Wiley Online、Web of Science、PubMed Central
- **中文平台 & 免费获取**：中国知网、Sci-Hub、Unpaywall

**分析历史（右侧边栏）**

- 自动记录每次分析，最多保存 50 条
- 无记录时显示占位提示，始终保持右侧布局完整
- 支持单条删除和一键清空
- 点击"查看"可重新打开历史分析结果

---

### 详情页（paper.html）

分析完成后自动跳转，展示以下内容：

- **AI 分析结果**：支持 Markdown 渲染（标题、粗体、列表、引用块等）
- **AI 思考过程**：如果模型返回了思考链（`<think>` 标签），以折叠形式展示
- **模板切换**：在详情页可随时切换模板重新生成，同一论文不同模板的结果会缓存到 localStorage
- **导出功能**：支持导出为 Markdown 文件和复制全文

---

### 设置页（settings.html）

**AI API 配置**

| 字段 | 说明 |
|------|------|
| Base URL | MiniMax API 端点。国际版：`https://api.minimax.chat/v1`；国内版：`https://api.minimaxi.com/v1` |
| API Key | 以 `eyJ` 开头的 Bearer Token，仅存储在本地 localStorage |
| 模型名称 | 例如 `MiniMax-Text-01`、`abab6.5s-chat`、`MiniMax-M2.5` 等 |

**自定义分析模板**

在文本框中输入自定义提示词，在详情页选择"✦ 自定义"模板时生效。

```
├── 说明.md              # 本文件

├── css/
│   ├── base.css         # CSS 变量与主题系统（亮色 / 暗色双主题）
│   ├── components.css   # 通用组件：导航栏、搜索栏、卡片、表单等
│   └── pages.css        # 页面级布局：首页两栏、历史区域、设置页等
│
└── js/
    ├── config.js        # localStorage 配置管理，支持点路径读写
    ├── main.js          # 主入口，三页面路由初始化
    ├── templates.js     # 五个 AI 分析模板的 systemPrompt 定义
    │
    ├── ai/
    │   ├── minimax.js   # MiniMax API 封装（OpenAI 兼容接口）
    │   └── modes.js     # 三种分析模式（A/B/C）+ pdf.js 懒加载
    │
    └── ui/
        ├── toast.js     # Toast 通知组件
        ├── detail.js    # 详情页渲染（Markdown 解析、思考链折叠、模板切换）
        ├── search.js    # 搜索栏组件（已保留供未来扩展）
        └── card.js      # 论文卡片组件（已保留供未来扩展）
```

## 技术实现

### 架构设计

- **纯静态**：零框架、零构建工具，可直接用浏览器打开 HTML 文件运行
- **模块化**：通过 `window.XXX` 暴露模块（替代 ES module，兼容本地文件协议）
- **无后端**：所有数据存储在 `localStorage`（配置 + 历史），分析结果通过 `sessionStorage` 在首页和详情页之间传递

### 核心流程

```
用户拖拽 PDF
    → pdf.js 提取文本（懒加载 CDN）
    → 截取前 14000 字符
    → MiniMax API（OpenAI 兼容格式）
    → 解析 <think> 标签（推理模型思考链）
    → 保存结果到 localStorage
    → sessionStorage 传递给 paper.html
    → Markdown 渲染展示
```

### AI 分析模式

| 模式 | 触发场景 | 说明 |
|------|----------|------|
| **模式 A** | PDF URL | 从外部 URL 下载并解析 PDF，存在 CORS 风险 |
| **模式 B** | 元数据 | 使用标题、摘要等结构化数据，无需 PDF |
| **模式 C** | 上传 PDF | 主力模式，本地解析，最准确 |

### 主题系统

CSS 变量驱动的双主题，通过 `data-theme` 属性切换：
- `data-theme="light"`（默认）：米白背景 + 低饱和靛蓝紫主色
- `data-theme="dark"`：深色背景 + 相同主色调

### Markdown 渲染

`detail.js` 内置轻量 Markdown 解析，支持：
- `【标题】` → 彩色分节标题（5 色轮换）
- `**粗体**` → `<strong>`
- `# Heading` / `## Heading` → `<h3>` / `<h4>`
- `• / - / * ` 开头 → 无序列表
- `> 引用` → 引用块
- `__THINK__....__ENDTHINK__` → 可折叠的 AI 思考过程

## 配置说明

所有配置存储在 `localStorage` 键 `papergainer_config` 中，结构如下：

```json
{
  "ai": {
    "provider": "minimax",
    "baseUrl": "https://api.minimax.chat/v1",
    "apiKey": "eyJ...",
    "model": "MiniMax-Text-01",
    "groupId": ""
  },
  "theme": "light",
  "searchHistory": [],
  "domains": [],
  "customPrompt": ""
}
```

分析历史单独存储在 `localStorage` 键 `pg_history`，最多 50 条，格式：

```json
[
  {
    "id": "1709123456789_abc12",
    "fileName": "paper.pdf",
    "mode": "C",
    "createdAt": 1709123456789,
    "results": {
      "structured": "【研究背景...】",
      "bullets": "**一句话总结**..."
    }
  }
]
```

## 常见问题

**Q：分析失败，提示"未配置 API Key"**
A：前往设置页填写 Base URL、API Key 和模型名称后保存。

**Q：PDF 内容提取为空或内容过少**
A：扫描件 PDF（图片型）无法提取文字，需要使用包含可选中文字的 PDF。

**Q：API 请求失败，提示 CORS 错误**
A：部分 API 服务商限制了浏览器直接调用。建议尝试切换 Base URL，或使用支持浏览器跨域请求的端点。

**Q：推理模型（如 MiniMax-M2.5）分析后出现一堆思考内容**
A：正常现象，思考过程会被自动识别并折叠展示，点击可展开查看。

**Q：历史记录丢失**
A：历史存储在浏览器 localStorage 中，清除浏览器数据会导致丢失。建议使用"导出 Markdown"功能保存重要分析结果。

**Q：可以在手机上使用吗**
A：可以，页面在窄屏下会切换为单栏布局，基本功能可正常使用，但体验以桌面为主。
